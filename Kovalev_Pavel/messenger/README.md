## 1. Описание

### Назначение
Программа представляет собой многопользовательский чат для локальной сети, написанный на C++ для Linux. Сервер работает на Raspberry Pi, а клиенты запускаются на Linux-машинах и подключаются к серверу по TCP. Помимо обмена сообщениями, сервер принимает данные от Arduino и выполняет мониторинг состояния системы.

### Основные возможности
- Поддержка нескольких клиентов одновременно.
- Обмен сообщениями в режиме broadcast.
- Отображение и хранение логов в памяти.
- Получение данных с Arduino по Serial.
- Контроль порогов датчиков и генерация предупреждений.
- Мониторинг RAM и температуры CPU Raspberry Pi.

## 2. Схема системы

### Общая архитектура
```text
┌──────────────────────────┐
│ CLIENT                   │
│ Linux                    │
│ client.cxx               │
│                          │
│ TCP :8080                │
└─────────────┬────────────┘
              │
              ▼
┌─────────────┴────────────┐
│ SERVER                   │
│ Raspberry Pi / Linux     │
│ server.cxx               │
│                          │
│ - чат                    │
│ - логирование            │
│ - мониторинг системы     │
│ - обработка Arduino      │
└─────────────┬────────────┘
              │
              ▼
┌─────────────┴────────────┐
│ ARDUINO                  │
│ arduino_logger.ino       │
│                          │
│ Serial 9600              │
│ A0, A1                   │
└──────────────────────────┘
```

### Логика взаимодействия
1. Arduino отправляет значения двух аналоговых датчиков в последовательный порт.
2. Сервер читает эти данные, обрабатывает их и формирует события логирования.
3. Клиенты подключаются к серверу по TCP.
4. Клиенты отправляют текстовые сообщения и команды.
5. Сервер рассылает сообщения подключенным клиентам.

## 4. Протокол сообщений

### Принцип обмена
Обмен данными выполняется через TCP в текстовом виде. Одно сообщение завершается символом новой строки `\n`. Сервер накапливает входящие данные в буфере, пока не получит полный текст сообщения.

### Формат сообщений клиента
Обычное сообщение:
```text
Привет всем
```

Команда:
```text
/users
```

### Формат сообщений сервера
Сообщение, пересланное сервером, имеет вид:
```text
Client <номер>: <текст сообщения>
```

Пример:
```text
Client 0: Привет всем
```

### Формат служебных сообщений
Сервер может отправлять:
- ответ на команду;
- историю логов;
- логи в режиме реального времени:
    - предупреждения;
    - показания датчиков;
    - системные уведомления о подключении Arduino.

Пример:
```text
Всего клиентов: 3
```

## 5. Работа с логами

### Назначение
Сервер ведёт журнал событий в памяти. Логи используются для отображения системных сообщений, событий Arduino, предупреждений и мониторинга состояния Raspberry Pi.

### Уровни логов
- `OFF`
- `INFO`
- `WARN`

### Что попадает в лог
- запуск сервера;
- подключение и отключение Arduino;
- данные от Arduino;
- предупреждение `Слишком близко!`;
- данные мониторинга системы;
- специальные команды клиента.

### Пример записи
```text
[INFO] 2026-04-15 13:35:00 Server started
```

### История логов
Команда `/logshistory` позволяет получить накопленные события из памяти сервера. Фильтрация поддерживается по уровню:
- без параметра — по умолчанию `INFO`;
- `warn` — только предупреждения и более важные события.

## 6. Мониторинг системы

### Что отслеживается
Сервер периодически получает:
- загрузку RAM в процентах;
- температуру CPU Raspberry Pi.

### Частота проверки
Мониторинг выполняется примерно каждые 10 секунд.

### Формат записи мониторинга
Пример:
```text
[INFO] 2026-04-15 13:35:10 Monitor: RAM 42% CPU temp 51 °C
```

### Назначение
Эта часть приложения позволяет наблюдать за состоянием самого хоста, на котором работает сервер. Информация выводится в журнал и может быть рассылаема клиентам, если у них включён соответствующий уровень отображения логов.

## 7. Датчики Arduino

### Назначение скетча
Скетч `arduino_logger.ino` считывает два аналоговых канала, к которым подключены инфракрасные датчики расстояния:
- `A0`
- `A1`

Значения выводятся в Serial с интервалом 500 мс.

### Формат данных
Arduino отправляет строку:
```text
<sensor_0>, <sensor_1>
```

Пример:
```text
512, 734
```

### Обработка на сервере
Сервер:
1. читает строку из Serial;
2. убирает лишние символы конца строки и пробелы;
3. разбивает строку по разделителю `, `;
4. преобразует значения в числа;
5. сравнивает их с порогом `700`.

### Условие предупреждения
Если хотя бы одно значение меньше `700`, сервер создаёт предупреждение:
```text
Слишком близко!
```

## 8. Добавление нового датчика

### Шаг 1. Arduino

Добавить пин в скетч:

```cpp
#define SENSOR_2_PIN A2
```

Чтение в `loop()`:

```cpp
void loop() {
  Serial.print(analogRead(SENSOR_0_PIN));
  Serial.print(", ");
  Serial.print(analogRead(SENSOR_1_PIN));
  Serial.print(", ");
  Serial.println(analogRead(SENSOR_2_PIN));
  i++;
  delay(500);
}
```

В `loop()` достаточно добавить разделитель и вывод данных. Последняя строка использует `println()` для переноса строки. 

### Шаг 2. Сервер (функция `handle_arduino_data`)

Обновить индексы:

```cpp
const int SENSOR_2_INDEX = 2; // в начеле файла

int sensor_0 = std::stoi(arduino_data_pieces[SENSOR_0_INDEX]);
int sensor_1 = std::stoi(arduino_data_pieces[SENSOR_1_INDEX]);
int sensor_2 = std::stoi(arduino_data_pieces[SENSOR_2_INDEX]);
```

Добавить проверку показний датчика:

```cpp
if (sensor_0 < 700 || sensor_1 < 700 || sensor_2 < 700) {
    logEvent(WARN, "Слишком близко!");
}
```

### Шаг 3. Проверка формата

Пример:
```text
512, 734, 623
```

### Шаг 4. Результат

При превышении порога на любом датчике:
```text
[INFO] 2026-04-15 13:35:00 Arduino: 512, 734, 623
[WARN] 2026-04-15 13:35:00 Слишком близко!
```

### Итог

Добавление датчика требует:
1. **Arduino** → добавить `#define`, чтение и вывод в Serial
2. **Сервер** → добавить константу индекса и проверку в `handle_arduino_data()`

## 9. Команды сервера

### Команды, обрабатываемые сервером
Сервер понимает следующие команды от клиента:
- `/users`
- `/logs [off/info/warn]`
- `/logshistory [info/warn]`
- `/p`

### `/users`
Возвращает количество подключённых клиентов.

Пример:
```text
Всего клиентов: 2
```

### `/logs [off/info/warn]`
Устанавливает уровень отображения логов для конкретного клиента.  
Пример:
```text
/logs warn
```

После этого клиент будет получать только логи уровня `WARN` и выше.

### `/logshistory [info/warn]`
Возвращает историю логов из памяти сервера с фильтрацией по уровню.

Пример:
```text
/logshistory warn
```

### `/p`
Позволяет клиенту сообщить о важной неполадке.

Создаёт событие в журнале:
```text
Пышки закончились
```

## 10. Команды клиента

### Команды, обрабатываемые локально
Клиент понимает команды:
- `/help`
- `/disconnect`

### `/help`
Показывает список доступных команд в консоли клиента.

### `/disconnect`
Завершает работу клиента и закрывает соединение с сервером.

### Остальные команды
Если сообщение начинается с `/`, оно считается командой. Если эта команда не обрабатывается локально, она отправляется на сервер для дальнейшей обработки.

### Обычные сообщения
Если строка не начинается с `/`, она считается обычным сообщением и отправляется на сервер.

## 11. Запуск

### Компиляция
Сборка клиентской и серверной части на Linux:
```bash
g++ server.cxx -o server
g++ client.cxx -o client
```

### Запуск сервера
На Raspberry Pi:
```bash
./server
```

Сервер начинает прослушивание порта `8080`, запускает поток мониторинга и поток чтения Arduino.

### Запуск клиента
На клиентской машине Linux:
```bash
./client
```

Клиент подключается к адресу сервера, указанному в коде, и открывает консольный интерфейс для сообщений.

### Arduino
Скетч `arduino_logger.ino` загружается в Arduino отдельно. После подключения к Raspberry Pi данные начинают поступать автоматически через последовательный порт.

***

## 12. Ограничения

### Технические ограничения
- Максимальное число клиентов: `10`.
- Адрес сервера в клиенте задан жёстко в коде.
- История логов хранится только в оперативной памяти.
- Протокол не содержит авторизации и шифрования.
- Обмен сообщениями основан на текстовом формате с разделением по `\n`.

### Поведение при сбоях
Если Arduino не отвечает более нескольких секунд, сервер считает устройство отключённым и пытается переподключиться.
