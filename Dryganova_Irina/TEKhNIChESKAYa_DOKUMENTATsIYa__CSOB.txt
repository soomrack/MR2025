1. Описание
CSOB — система удаленного управления роботом (Omegabot) через Wi-Fi с использованием Raspberry Pi в качестве сервера-посредника. Сервер принимает TCP-подключения на порту 8888, передает команды в Arduino по USB и логирует все события в файл robot.log. Предусмотрены два типа клиентов: клиент управления (robot_client) и клиент запроса логов (client_logger).

Эта система позволяет клиенту (robot_client.cpp) с компьютера Windows управлять движением робота (вперед, назад, повороты, остановка, регулировка скорости) с помощью нажатия клавиш WASD, при этом все команды передаются через сервер-посредник Raspberry Pi на Arduino Uno, который управляет моторами. Система обеспечивает логирование всех событий (подключения клиентов, отправленные команды, подтверждения от Arduino) в файл robot.log и предоставляет возможность отдельному клиенту (client_logger.cpp) запрашивать историю логов за указанный период (минуты, часы, дни), автоматически сохраняя её в текстовый файл и открывая в Блокноте.

4. Управление роботом

Команды клиента (robot_client):

  w - вперед (выводит:CMD:w OK)
  s - назад (выводит:CMD:s OK)
  d - вправо (выводит:CMD:d OK)
  a - влево (выводит:CMD:a OK)
  x - стоп (выводит:CMD:x OK)
  speed N - установка скорости (0-255) (выводит: SPEED: N)

  Клиент работает в режиме неблокирующего ввода: команды отправляются сразу при нажатии клавиши, без ожидания Enter.

Команды USB (RPi → Arduino):
  w - оба мотора вперед: drive(+spd, +spd)
  s - оба мотора назад: drive(-spd, -spd)
  d - левый назад, правый вперед: (-spd, +spd)
  a - левый вперед, правый назад: (+spd, -spd)
  x - stopMotors()
  speed N - currentSpeed = N (0-255) 
  
  Каждая команда движения подтверждается ответом CMD:x OK (где x — команда).

6. Установка
Сервер (Raspberry Pi 3B+):

  Подключение по: ssh
 
  Загрузка: 
  cd /home/pi
  nano server_logger.cpp
  
  Компиляция:
  g++ -std=c++11 -o server_logger server_logger.cpp -lpthread

  Запуск:
  ./server_logger
  
  При успешном запуске появится:
    === Robot Server ===
    Connecting to /dev/ttyACM0...
    Port opened successfully
    Server started on port 8888
    IP addresses:
      - 192.168.1.100
    Waiting for clients...

  Клиенты (Windows, Visual Studio)
   Для каждого клиента (robot_client и client_logger):
   1. Открыть Visual Studio 
   2. Создать новый проект: Файл → Создать → Проект → Консольное приложение C++
   3. Вставить код
   4. Собрать: Ctrl+Shift+B
   5. Запустить: Ctrl+F5

  Для robot_client:
    - Ввести IP сервера
    - Управлять клавишами W, A, S, D, X, Q

  Для client_logger:
    - Ввести IP сервера
    - Вводить запросы вида 10m, 2h, 3d
    - Q для выхода

  Arduino Uno
    1. Открыть car_logger.ino в Arduino IDE
    2. Выбрать плату: Инструменты → Плата → Arduino Uno
    3. Выбрать порт: Инструменты → Порт → COM...
    4. Нажать "Загрузить"
    5. Проверка: после загрузки открыть Serial Monitor:
     - должно появиться ARD:READY
     - при отправке w — ответ CMD:w OK

7. Тестирование

Проверка USB-связи без клиента:
  На Raspberry Pi:
  1. Узнаем какой порт: cat /dev/tty*
  2. В зависимости от найденного порта:
      - cat /dev/ttyACM0
      - cat /dev/ttyUSB0
  Должны появляться строки ARD:READY (после перезагрузки Arduino)

Проверка команд мотора:
  Через robot_client:
  - подключиться, нажать W — моторы должны крутиться вперед.
    На сервере в логе:
    [CMD] Client 192.168.1.50 -> w
    [USB] -> Arduino: w
    [ARDUINO] Arduino -> CMD:w OK

Проверка запроса истории:
  1. Запустить client_logger.exe
  2. Ввести IP, затем "1m" (последняя минута)
  3. Должен появиться файл log_...txt с содержимым:
    === Log for last 1 min (X records) ===
    ...

Проверка таймаута Arduino:
  1. Запустить сервер
  2. Отправить команду W с клиента
  3. Отключить USB-кабель Arduino
  4. Снова отправить команду — сервер должен вернуть ERROR: Arduino not responding

Автоостановка при потере связи
  На Arduino установлен TIMEOUT = 300 мс:
  если команды не поступают более 300 мс, stopMotors() останавливает моторы.
  Проверка: отправить W, затем отключить сервер — через 300 мс моторы остановятся.
