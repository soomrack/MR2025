\subsection{Шейдеры OpenGL~3.3}
 
Визуализатор использует две пары GLSL-шейдеров. Первая
(\texttt{VERT3D}/) применяется для рендеринга 3D-модели
полётного контроллера с диффузным освещением.
 
Вторая пара (\texttt{VERT\_FLAT}/\texttt{FRAG\_FLAT}) используется для
отрисовки всех линейных примитивов без освещения: координатной сетки,
мировых осей, траектории и HUD-графиков.

\subsection{Процедурная модель полётного контроллера}
 
3D-модель FC строится процедурно из параллелепипедов функцией
\texttt{build\_fc()}. Вспомогательная \texttt{push\_box()} генерирует
шесть граней куба с различными коэффициентами яркости для имитации объёма;
\texttt{push\_arrow()} добавляет три цветных стрелки осей (X~--- красная,
Y~--- зелёная, Z~--- синяя):
 
\begin{lstlisting}[caption={Процедурная сборка модели FC}]
static std::vector<V3> build_fc() {
    std::vector<V3> v;
    // Корпус -- тёмно-серый прямоугольник
    push_box(v,  0,     0,    0,    0.8f,0.8f,0.14f, 0.13f,0.14f,0.19f);
    // Белая полоска "нос" (направление Y+)
    push_box(v,  0,  0.37f, 0.08f, 0.7f,0.06f,0.04f, 0.95f,0.95f,0.95f);
    // Боковые рёбра
    push_box(v,  0.37f, 0,  0.08f, 0.04f,0.8f,0.04f, 0.22f,0.22f,0.28f);
    push_box(v, -0.37f, 0,  0.08f, 0.04f,0.8f,0.04f, 0.22f,0.22f,0.28f);
    // Стрелки осей
    push_arrow(v, 0,0,0.1f,  1,0,0, 0.7f, 0.95f,0.10f,0.10f); // X
    push_arrow(v, 0,0,0.1f,  0,1,0, 0.7f, 0.10f,0.92f,0.10f); // Y
    push_arrow(v, 0,0,0.07f, 0,0,1, 0.7f, 0.15f,0.42f,0.98f); // Z
    return v;
}
\end{lstlisting}
 
\subsection{Главный цикл рендера}
 
Функция \texttt{run\_visualizer()} инициализирует контекст GLFW/GLEW,
компилирует шейдеры, загружает геометрию в видеопамять и запускает главный
цикл. Схема каждого кадра:
\begin{enumerate}[leftmargin=2cm]
    \item Обработать события ввода (мышь, клавиши).
    \item Вычитать из \texttt{SharedBuffer} \textit{все} накопившиеся
          пакеты, для каждого вызвать \texttt{integrate\_imu()}, добавить
          точку траектории и обновить кольцевые буферы.
    \item Обновить матрицы орбитальной камеры, следящей за FC.
    \item Отрисовать \textbf{3D-сцену}:
          сетку, мировые оси, траекторию с цветовым затуханием, модель FC.
    \item Отрисовать \textbf{HUD-панель}:
          четыре графика временных рядов.
\end{enumerate}
 
Матрица модели FC формируется из текущей позиции и кватерниона ориентации с
помощью GLM:
 
\begin{lstlisting}[caption={Матрица модели из позиции и кватерниона}]
glm::mat4 fc_pos = glm::translate(glm::mat4(1),
    glm::vec3((float)nav.px, (float)nav.py, (float)nav.pz));
glm::quat  q((float)nav.qw, (float)nav.qx,
             (float)nav.qy, (float)nav.qz);
glm::mat4 fc_model = fc_pos * glm::mat4_cast(q);
glm::mat4 fc_mvp   = vp * fc_model;
\end{lstlisting}
 
Траектория отрисовывается как набор отрезков с цветовым градиентом: старые
точки темнее, новые~--- ярче.
 
\subsection{HUD-панель графиков}
 
Нижняя панель содержит четыре независимых графика на основе скользящего
буфера \texttt{RingBuf} (300~последних значений). Каждый график отображает
три сигнала (по осям X, Y, Z) красным, зелёным и синим цветами:
