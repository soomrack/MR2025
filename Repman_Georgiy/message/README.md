# Chat MAX 228.0 – кроссплатформенный чат + управление роботом

Многопользовательский чат-сервер с возможностью дистанционного управления гусеничным роботом через UART (Arduino Mega 2560).  
Работает на **Linux** (Raspberry Pi, Ubuntu) и **Windows** (MinGW).

## Особенности

- **Кроссплатформенность** – сервер и клиент работают на Linux и Windows.
- **Многопользовательский чат** – до 10 клиентов, цветные имена, эмодзи-шорткаты.
- **Управление роботом** – WASD-режим с raw‑вводом (мгновенная реакция на клавиши).
- **Автоматическое переподключение** клиента при обрыве связи.
- **Логирование** – системная статистика (CPU/RAM) и ток двигателей.
- **Поддержка UART** – на Linux через `/dev/ttyAMA0`, на Windows через `COM3` (настраивается).

## Требования

### Общие
- Компилятор с поддержкой C99 (gcc, mingw-gcc)
- Make (опционально)

### Linux
- Установленный `build-essential`
- Права на чтение/запись UART-порта (`/dev/ttyAMA0` или другой)

### Windows
- **MinGW-w64** (с поддержкой pthreads, например из [msys2](https://www.msys2.org/))
- Подключённый USB‑UART адаптер (COM-порт)
- Библиотека `ws2_32` (линкуется автоматически через `-lws2_32`)

## Сборка

### Linux

```bash
# Установка компилятора
sudo apt install build-essential

# Переход в папку с исходниками
cd message

# Сборка
make

# Будут созданы исполняемые файлы:
#   chat_server   (сервер)
#   chat_client   (клиент)

Windows (MinGW)

Откройте командную строку MinGW или обычную CMD (если MinGW добавлен в PATH).
bash

cd message

# Сборка через make
mingw32-make

# Или вручную:
gcc server.c motor.c -o chat_server.exe -lws2_32
gcc client.c -o chat_client.exe -lws2_32

Примечание: Если mingw32-make не работает, используйте make (в зависимости от установки).
Makefile автоматически определяет Windows и добавляет флаг -D_WIN32 и библиотеку ws2_32.
Запуск
Сервер

Linux:
bash

./chat_server

Windows:
bash

chat_server.exe

При успешном запуске вы увидите приветствие и информацию о порте.
Если UART недоступен (нет устройства или COM-порта), сервер запустится без поддержки моторов, чат будет работать.
Клиент

Linux / Windows:
bash

./chat_client        # Linux
chat_client.exe      # Windows

По умолчанию клиент подключается к 127.0.0.1:5000.
Чтобы подключиться к удалённому серверу, измените IP в файле client.c (строка с inet_pton) и пересоберите клиент.
Настройка UART / COM-порта
Linux (Raspberry Pi)

    Включите последовательный интерфейс:
    bash

    sudo raspi-config
    # Interface Options → Serial Port → login shell over serial → NO
    #                                 → serial port hardware → YES

    Добавьте пользователя в группу dialout:
    bash

    sudo usermod -aG dialout $USER
    # перелогиньтесь

    Убедитесь, что устройство существует: ls -l /dev/ttyAMA0

Windows

    Подключите USB-UART адаптер и узнайте номер COM-порта (Диспетчер устройств → Порты (COM и LPT)).

    Откройте motor.h и измените строку:
    c

    #define MOTOR_UART_PORT   "COM3"   // замените на ваш COM-порт

    Пересоберите сервер.

Прошивка Arduino

Файл motor_arduino.ino предназначен для Arduino Mega 2560.

    Откройте файл в Arduino IDE.

    Выберите плату: Tools → Board → Arduino Mega 2560.

    Выберите порт (обычно USB).

    Загрузите прошивку.

    Убедитесь, что настройки скорости в коде совпадают с сервером (115200 бод).

Пины подключения моторов и датчиков тока:
Мотор	PWM	INA	INB	EN	Ток
Левый	6	7	30	31	A0
Правый	5	32	4	33	A1

Управление роботом
Команды для входа в режим управления
Команда	Действие
\drive	Войти в WASD-режим (raw terminal)
\drive_speed N	Установить скорость мотора (0–100)
\drive_key X	Отправить команду движения (используется автоматически)
Управление в WASD-режиме

После ввода \drive клиент переключается в raw-режим.
Клавиши управления:

    w – вперёд

    s – назад

    a – поворот налево

    d – поворот направо

    пробел – стоп

    q – выход из режима (моторы останавливаются)

Скорость задаётся заранее через \drive_speed 70.
Перед каждой командой движения сервер автоматически отправляет Arduino текущую скорость.
Логирование мотора

Данные о токе (CURR:left,right) и предупреждения о перегрузке (>3.0 А) пишутся в motor.log.

Пример:
text

[2026-04-06 23:09:18] SERVER: INIT — motors stopped
[2026-04-06 23:09:53] Artem: ENTERED DRIVE MODE
[2026-04-06 23:09:54] Artem: KEY 'w' -> FORWARD speed=50%
[2026-04-06 23:09:55] INFO: Current: left=1.23A right=1.18A
