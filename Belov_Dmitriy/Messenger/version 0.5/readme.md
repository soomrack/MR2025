

# Почему в проекте так много файлов

В данной работе клиент и сервер разделены на несколько файлов (`.h` и `.cpp`), хотя технически всё можно было бы написать в одном файле. Такое разделение сделано осознанно, чтобы программа была более понятной, удобной для расширения и поддержки.


Файл с расширением `.h` (header) — это заголовочный файл.  
В нём обычно описываются классы, структуры, перечисления и объявления функций.


Такой подход является стандартной практикой в C++.

## Зачем разделять код

В проекте выделены отдельные части:

- `Message.h` — описывает формат сообщения (тип и размер данных).
- `SocketBase.h` — содержит общую логику работы с сокетом.
- `Client.h` — описывает клиент.
- `Server.h` — описывает сервер.

Каждый файл отвечает за свою задачу.
Если всё написать в одном файле, код быстро становится громоздким и трудным для понимания.

## Как это помогает расширяемости

Структура проекта позволяет добавлять новые возможности.

Например, если нужно добавить передачу изображений или видео:

- в `Message.h` можно добавить новый тип сообщения;
- в `Client` добавить функцию отправки изображения;
- в `Server` добавить обработку нового типа данных.

При этом не нужно переписывать весь проект.

Если позже понадобится:
- шифрование,
- сжатие данных,
- поддержка нескольких клиентов,
- работа с файлами,

то изменения можно будет вносить в конкретные части программы, не затрагивая всё остальное.
