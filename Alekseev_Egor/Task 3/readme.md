СИСТЕМА УДАЛЕННОГО МОНИТОРИНГА СЕНСОРНЫХ ДАННЫХ
================================================

ОБЩЕЕ ОПИСАНИЕ
================================================
Система представляет собой клиент-серверное приложение для удаленного 
мониторинга показателей сенсоров (температура, влажность воздуха, влажность почвы, 
освещенность) в реальном времени. Сервер считывает данные с Arduino через UART, 
логирует их и предоставляет клиентам по TCP протоколу.

ПОДКЛЮЧЕНИЕ ДАТЧИКОВ К ARDUINO
================================================

Схема подключения:
------------------
Arduino Uno

Датчик DHT11 (температура и влажность):
  VCC  -> 5V
  GND  -> GND
  DATA -> Pin 2

Датчик влажности почвы (аналоговый):
  VCC  -> 5V
  GND  -> GND
  AOUT -> A3 (Pin 3)

Датчик освещенности (аналоговый):
  VCC  -> 5V
  GND  -> GND
  AOUT -> A4 (Pin 4)

Примечание: Для точных измерений рекомендуется использовать 
делитель напряжения с фоторезистором и резистором 10кОм.


ОПИСАНИЕ РАБОТЫ ARDUINO
================================================

Принцип работы:
--------------
1. Arduino ожидает команду по Serial порту
2. При получении символа '1' считывает показания со всех датчиков
3. Конвертирует сырые значения в удобный формат
4. Отправляет данные в формате: "температура влажность освещенность влажность_почвы"
5. Возвращается в режим ожидания следующей команды

Формат передаваемых данных:
--------------------------
"[temperature] [humidity] [light] [soil_moisture]"

Пример: "23 45 550 65"

Где:
- temperature: температура в градусах Цельсия (int)
- humidity: влажность воздуха в процентах (int)
- light: освещенность в люксах (int, 0-800)
- soil_moisture: влажность почвы в процентах (int, 0-100)

Библиотеки Arduino:
------------------
- DHT.h - библиотека для работы с датчиком DHT11/DHT22
  Установка: Sketch -> Include Library -> Manage Libraries -> "DHT sensor library"


СЕРВЕРНАЯ ЧАСТЬ

НАЗНАЧЕНИЕ
----------
Серверное приложение выполняет следующие функции:
- Считывание данных с Arduino через последовательный порт (UART)
- Валидация и анализ данных на предмет выхода за допустимые диапазоны
- Логирование всех данных и опасных событий в файлы
- Обработка подключений клиентов по TCP/IP
- Предоставление данных клиентам в реальном времени

----------------------
- ОС: Linux (разработано для Raspberry Pi/Ubuntu)
- Компилятор: GCC с поддержкой C++11 или выше
- Библиотеки: pthread, стандартная библиотека C++
- Права доступа: доступ к последовательному порту (/dev/ttyACM0)
- Сетевые настройки: открытый порт 8080

ОСНОВНЫЕ КОМПОНЕНТЫ СЕРВЕРА
---------------------------

ArduinoUART
- Класс для взаимодействия с Arduino через последовательный порт
- Методы:
  * connect() - установка соединения с Arduino
  * disconnect() - разрыв соединения
  * readSensorData() - чтение данных с датчиков
  * isConnected() - проверка состояния соединения

RealSensorData
- Обертка для получения актуальных данных с сенсоров
- Методы:
  * getSensorData() - получение новых данных

DataLogger
- Система логирования данных сенсоров
- Методы:
  * stop() - остановка логирования
  * getLastEntries(count) - получение последних N записей
  * getLastDangerEntries(count) - получение опасных событий
  * getTotalEntries() - общее количество записей
  * getTotalDangerEntries() - количество опасных событий

CommandHandler
- Обработчик команд от клиентов

ClientHandler
- Обработчик клиентских подключений

ПОДДЕРЖИВАЕМЫЕ КОМАНДЫ СЕРВЕРА
------------------------------
GET_DATA        - Получить текущие показания сенсоров
START_STREAM    - Начать потоковую передачу (каждые 2 сек)
STOP_STREAM     - Остановить потоковую передачу
GET_LOG         - Получить последние 120 записей лога
GET_DANGER_LOG  - Получить последние 120 опасных событий
QUIT            - Отключиться от сервера
SHUTDOWN        - Завершить работу сервера


АЛГОРИТМ ОБМЕНА ДАННЫМИ С ARDUINO
---------------------------------
1. Сервер отправляет байт '1' в UART
2. Ожидает 100 мс для обработки запроса Arduino
3. Считывает ответ из UART (до 256 байт)
4. Парсит строку с данными в формате "temp humidity light soil_moisture"
5. Если данные не получены, возвращает нулевые значения
6. При ошибке чтения выводит диагностическое сообщение

УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ СЕРВЕРА
---------------------------------
Ошибка подключения к Arduino:
  [FATAL] Failed to connect to Arduino on /dev/ttyACM0
  Решение: 
    1. Проверить подключение Arduino: ls -la /dev/ttyACM*
    2. Выдать права: sudo chmod 666 /dev/ttyACM0

Нет данных от Arduino:
  [Warning] No valid data from Arduino yet
  Решение: 
    1. Проверить подключение датчиков
    2. Проверить работу Arduino через Serial Monitor
    3. Убедиться, что скорость UART совпадает (9600)

Ошибка парсинга данных:
  [UART] Failed to parse data: 'некорректные данные'
  Решение: 
    1. Проверить формат вывода Arduino
    2. Убедиться в отсутствии лишних символов в Serial.println

Ошибка bind порта:
  Bind failed: Address already in use
  Решение: Подождать освобождения порта или изменить SERVER_PORT

НАЗНАЧЕНИЕ
----------
Клиентское приложение для Windows, обеспечивающее:
- Подключение к серверу по TCP
- Отправка команд серверу
- Получение и отображение данных сенсоров
- Поддержка потокового режима отображения данных
- Интерактивный ввод команд


ТРЕБОВАНИЯ К ОКРУЖЕНИЮ
----------------------
- ОС: Windows 7/10/11
- Компилятор: Visual Studio или MinGW с поддержкой C++11
- Библиотеки: Winsock2 (Ws2_32.lib)


КОНФИГУРАЦИЯ КЛИЕНТА
--------------------
SERVER_PORT = 8080                  # Порт сервера
SERVER_ADDRESS = "192.168.1.56"     # IP-адрес сервера


ОСНОВНЫЕ КОМПОНЕНТЫ КЛИЕНТА
---------------------------

CommandHandler
- Обработчик команд клиента с поддержкой локальных и серверных команд

Локальные команды (выполняются на клиенте):
  HELP - Показать список доступных команд
  QUIT - Отключиться и выйти
  SHUTDOWN - Выйти и остановить сервер

Серверные команды (передаются на сервер):
  GET_DATA - Запросить данные сенсоров
  START_STREAM - Начать потоковую передачу
  STOP_STREAM - Остановить потоковую передачу
  GET_LOG - Запросить историю измерений
  GET_DANGER_LOG - Запросить историю опасных событий

----------------------
Пример работы:
  === Chat started ===
  Enter HELP for available commands
  > HELP

  === Available commands ===
    HELP - Show available commands
    GET_DATA - Get sensor's data
    START_STREAM - Start streaming sensor data every 2 seconds

ПРОТОКОЛ ОБМЕНА ДАННЫМИ
================================================

ПОЛНАЯ ЦЕПОЧКА ПЕРЕДАЧИ ДАННЫХ
------------------------------
1. Клиент отправляет команду (например, GET_DATA) на сервер по TCP
2. Сервер получает команду и отправляет байт '1' в Arduino по UART
3. Arduino считывает данные с датчиков и отправляет их обратно по UART
4. Сервер получает данные, форматирует их и отправляет клиенту по TCP
5. Клиент отображает полученные данные пользователю

ФОРМАТ КОМАНД
-------------
Команды передаются как текстовые строки в верхнем регистре:
  GET_DATA
  START_STREAM
  QUIT

ФОРМАТ ОТВЕТОВ
--------------
Обычный ответ:
  Temperature: 23.5°C, Humidity: 45.2%, Soil Moisture: 65.0%, Light: 550.0 lux

Потоковые данные:
  DATA: Temperature: 23.5°C, Humidity: 45.2%, Soil Moisture: 65.0%, Light: 550.0 lux

РАЗВЕРТЫВАНИЕ СИСТЕМЫ
================================================

ПОДГОТОВКА ARDUINO
------------------
1. Установить Arduino IDE
2. Установить библиотеку DHT sensor library:
   Sketch -> Include Library -> Manage Libraries -> Поиск "DHT sensor library"
3. Подключить датчики согласно схеме подключения
4. Загрузить скетч sensor_arduino.ino в Arduino
5. Проверить работу через Serial Monitor (9600 бод)
   Отправить '1' - должен прийти ответ с данными сенсоров


ПОДГОТОВКА СЕРВЕРА
------------------
1. Подключить Arduino к USB порту
2. Проверить, что Arduino определился как /dev/ttyACM0:
   ls -la /dev/ttyACM*
3. Если порт другой, изменить UART_PORT в исходном коде сервера
4. Скомпилировать серверную часть:
   g++ -std=c++11 -pthread sensor_server.cpp -o sensor_server
5. Настроить права для UART:
   sudo chmod 666 /dev/ttyACM0
6. Запустить сервер:
   ./sensor_server


ПОДГОТОВКА КЛИЕНТА
------------------
1. Скомпилировать клиентскую часть в Visual Studio или MinGW:
   g++ -std=c++11 client.cpp -o Client.exe -lws2_32
2. Настроить IP-адрес сервера в коде (константа SERVER_ADDRESS)
3. Запустить клиент:
   Client.exe


ПРОВЕРКА РАБОТОСПОСОБНОСТИ
--------------------------
1. Arduino подключен и работает (проверить через Serial Monitor)
2. Сервер запущен и показывает:
   [UART] Connected to Arduino on /dev/ttyACM0
   [System] Successfully received data from Arduino!
3. Клиент подключен и показывает:
   Successfully connected to server!
4. Отправить команду GET_DATA - должны отобразиться показания сенсоров

РАСШИРЕНИЕ СИСТЕМЫ
================================================

ДОБАВЛЕНИЕ НОВЫХ СЕНСОРОВ
-------------------------
Для добавления нового датчика необходимо:

1. На стороне Arduino:
   - Подключить датчик к свободному пину
   - Добавить чтение в readSensors()
   - Добавить конвертацию в convertData()
   - Добавить значение в строку вывода send_readings()

2. На стороне сервера:
   - Добавить поле в структуру SensorData
   - Обновить парсинг в ArduinoUART::readSensorData()
   - Обновить формат вывода в методах SensorData
   - Добавить проверки в SensorData::checkDangerous()
   - Добавить диапазоны в NormalRanges

3. На стороне клиента:
   - Клиент не требует изменений при добавлении сенсоров
   - Данные будут отображаться автоматически


ДОБАВЛЕНИЕ НОВЫХ КОМАНД
-----------------------
1. Зарегистрировать команду в конструкторе CommandHandler (сервер)
2. Зарегистрировать обработчик в конструкторе CommandHandler (клиент)
3. Реализовать логику обработки на серверной стороне

Пример добавления команды PING:
  Сервер:
    registerCommand("PING", "Check server connection", CommandType::SERVER,
        [this](int sock, const std::string& cmd) -> bool {
            sendResponse(sock, "PONG");
            return true;
        });
  
  Клиент:
    registerCommand("PING", "Check server connection", CommandType::SERVER,
        [this](int sock, const std::string& cmd) -> bool {
            send(sock, cmd.c_str(), cmd.length(), 0);
            return true;
        });
