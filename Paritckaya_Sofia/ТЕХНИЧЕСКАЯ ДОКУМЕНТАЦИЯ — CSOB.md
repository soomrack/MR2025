1\. Описание

CSOB — система удаленного управления роботом (Omegabot) через Wi-Fi с использованием Raspberry Pi в качестве сервера-посредника. Сервер принимает TCP-подключения на порту 8888, передает команды в Arduino по USB и логирует все события в файл robot.log. Предусмотрены два типа клиентов: клиент управления (robot\_client) и клиент запроса логов (client\_logger).

RIGHT\_DIR\_PIN  4 - направление правого мотора

4\. Управление роботом



Команды клиента (robot\_client):



&#x20; w - вперед (выводит:CMD:w OK)

&#x20; s - назад (выводит:CMD:s OK)

&#x20; d - вправо (выводит:CMD:d OK)

&#x20; a - влево (выводит:CMD:a OK)

&#x20; x - стоп (выводит:CMD:x OK)

&#x20; speed N - установка скорости (0-255) (выводит: SPEED: N)



&#x20; Клиент работает в режиме неблокирующего ввода: команды отправляются сразу при нажатии клавиши, без ожидания Enter.



Команды USB (RPi → Arduino):

&#x20; w - оба мотора вперед: drive(+spd, +spd)

&#x20; s - оба мотора назад: drive(-spd, -spd)

&#x20; d - левый назад, правый вперед: (-spd, +spd)

&#x20; a - левый вперед, правый назад: (+spd, -spd)

&#x20; x - stopMotors()

&#x20; speed N - currentSpeed = N (0-255) 

&#x20; 

&#x20; Каждая команда движения подтверждается ответом CMD:x OK (где x — команда).

6\. Установка

Сервер (Raspberry Pi 3B+):



&#x20; Подключение по: ssh

&#x20;

&#x20; Загрузка: 

&#x20; cd /home/pi

&#x20; nano server\_logger.cpp

&#x20; 

&#x20; Компиляция:

&#x20; g++ -std=c++11 -o server\_logger server\_logger.cpp -lpthread



&#x20; Запуск:

&#x20; ./server\_logger

&#x20; 

&#x20; При успешном запуске появится:

&#x20;   === Robot Server ===

&#x20;   Connecting to /dev/ttyACM0...

&#x20;   Port opened successfully

&#x20;   Server started on port 8888

&#x20;   IP addresses:

&#x20;     - 192.168.1.100

&#x20;   Waiting for clients...



&#x20; Клиенты (Windows, Visual Studio)

&#x20;  Для каждого клиента (robot\_client и client\_logger):

&#x20;  1. Открыть Visual Studio 

&#x20;  2. Создать новый проект: Файл → Создать → Проект → Консольное приложение C++

&#x20;  3. Вставить код

&#x20;  4. Собрать: Ctrl+Shift+B

&#x20;  5. Запустить: Ctrl+F5



&#x20; Для robot\_client:

&#x20;   - Ввести IP сервера

&#x20;   - Управлять клавишами W, A, S, D, X, Q



&#x20; Для client\_logger:

&#x20;   - Ввести IP сервера

&#x20;   - Вводить запросы вида 10m, 2h, 3d

&#x20;   - Q для выхода



&#x20; Arduino Uno

&#x20;   1. Открыть car\_logger.ino в Arduino IDE

&#x20;   2. Выбрать плату: Инструменты → Плата → Arduino Uno

&#x20;   3. Выбрать порт: Инструменты → Порт → COM...

&#x20;   4. Нажать "Загрузить"

&#x20;   5. Проверка: после загрузки открыть Serial Monitor:

&#x20;    - должно появиться ARD:READY

&#x20;    - при отправке w — ответ CMD:w OK



7\. Тестирование



Проверка USB-связи без клиента:

&#x20; На Raspberry Pi:

&#x20; 1. Узнаем какой порт: cat /dev/tty\*

&#x20; 2. В зависимости от найденного порта:

&#x20;     - cat /dev/ttyACM0

&#x20;     - cat /dev/ttyUSB0

&#x20; Должны появляться строки ARD:READY (после перезагрузки Arduino)



Проверка команд мотора:

&#x20; Через robot\_client:

&#x20; - подключиться, нажать W — моторы должны крутиться вперед.

&#x20;   На сервере в логе:

&#x20;   \[CMD] Client 192.168.1.50 -> w

&#x20;   \[USB] -> Arduino: w

&#x20;   \[ARDUINO] Arduino -> CMD:w OK



Проверка запроса истории:

&#x20; 1. Запустить client\_logger.exe

&#x20; 2. Ввести IP, затем "1m" (последняя минута)

&#x20; 3. Должен появиться файл log\_...txt с содержимым:

&#x20;   === Log for last 1 min (X records) ===

&#x20;   ...



Проверка таймаута Arduino:

&#x20; 1. Запустить сервер

&#x20; 2. Отправить команду W с клиента

&#x20; 3. Отключить USB-кабель Arduino

&#x20; 4. Снова отправить команду — сервер должен вернуть ERROR: Arduino not responding



Автоостановка при потере связи

&#x20; На Arduino установлен TIMEOUT = 300 мс:

&#x20; если команды не поступают более 300 мс, stopMotors() останавливает моторы.

&#x20; Проверка: отправить W, затем отключить сервер — через 300 мс моторы остановятся.
